# 設定画面設計

---

## 画面構成

```
┌─────────────────────────────┐
│  設定                       │
├─────────────────────────────┤
│  購入                       │
│    広告を削除               │  → 買い切り課金（StoreKit）
│    購入を復元               │
├─────────────────────────────┤
│  キャッシュ                 │
│    サムネイルキャッシュ      │
│    容量上限    [500MB >]    │  → 100MB / 250MB / 500MB / 1GB から選択
│    キャッシュをクリア       │  → 確認アラート後に実行
├─────────────────────────────┤
│  再生                       │
│    動画の先読み  [ON >]     │  → ON / OFF
│    デフォルト並び替え [名前順 >] │  → 並び替えキーを選択
├─────────────────────────────┤
│  データ                     │
│    視聴履歴をすべて削除     │  → 確認アラート後に実行
│    メディアリストをすべて削除 │  → 確認アラート後に実行
├─────────────────────────────┤
│  情報                       │
│    バージョン        1.0.0  │  → タップ不可
│    OSSライセンス            │  → ライセンス一覧画面へ
│    プライバシーポリシー     │  → Webビューで表示
│    利用規約                 │  → Webビューで表示
└─────────────────────────────┘
```

## 各項目の仕様

### 広告を削除

- StoreKit 2 を使用した買い切り課金
- 購入済みの場合はグレーアウトして「購入済み」と表示
- 購入フロー: タップ → App Store標準の購入確認 → 完了後に広告非表示

### 購入を復元

- デバイス変更・再インストール後に購入状態を復元
- `StoreKit.Transaction.currentEntitlements` で確認

### サムネイルキャッシュ容量上限

- デフォルト: 500MB
- 選択肢: 100MB / 250MB / 500MB / 1GB
- 上限を下げた場合、超過分をその場でLRU削除

### キャッシュをクリア

- 確認アラート: 「サムネイルキャッシュをすべて削除しますか？次回表示時に再生成されます」
- 実行後、`Library/Caches/thumbnails/` 以下を全削除
- サムネイルは次回ディレクトリ表示時に自動再生成されるため、ユーザーへの影響は最小限

### 動画の先読み

- ON（デフォルト）: `PrefetchManager` を有効化
- OFF: 先読みを無効化。モバイルデータ通信時の節約などに

### デフォルト並び替え

- 全フォルダ共通のデフォルト並び替えキーを設定
- 個別フォルダで変更した場合はフォルダ固有の設定が優先される（既存の仕様通り）

### 視聴履歴・メディアリストの全削除

- 各確認アラート: 「すべての視聴履歴を削除しますか？この操作は取り消せません」
- SwiftDataから該当モデルを全件削除

### OSSライセンス

VLCKit（LGPLv2.1）の表示が必須。他に使用するライブラリも合わせて一覧表示。

```
OSSライセンス
  ├── VLCKit - LGPLv2.1
  ├── AMSMB2 - MIT
  └── Google Mobile Ads SDK - 独自ライセンス
```

## 設定値の永続化

`UserDefaults` に保存する。

| キー | 型 | デフォルト値 |
|---|---|---|
| `thumbnailCacheLimitMB` | Int | 500 |
| `prefetchEnabled` | Bool | true |
| `defaultSortKey` | String | `nameAsc` |
| `adsRemoved` | Bool | false（StoreKitで管理、UserDefaultsはキャッシュ用） |
