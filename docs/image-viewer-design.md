# 画像ビューアー UI 設計

---

## 基本動作

動画プレイヤーと同様に `fullScreenCover` で全画面表示する。
タップでコントロールを表示・非表示。数秒操作がなければ自動的にフェードアウト。

連番画像を主体とするフォルダでも、まずは通常のディレクトリ表示を行う。
誤爆を避けるため、フォルダ遷移時に漫画ビューアへ自動遷移する仕様は採用しない。

---

## レイアウト構成

```
┌─────────────────────────────────────┐
│ ←   ファイル名.jpg        [共有]   │  ← トップバー（タップで表示）
├─────────────────────────────────────┤
│                                     │
│                                     │
│           画像表示エリア             │
│        （ピンチズームあり）          │
│                                     │
│                                     │
├─────────────────────────────────────┤
│  ◀                            ▶    │  ← 前後ファイルへの矢印ボタン
└─────────────────────────────────────┘
```

---

## 操作

| 操作 | 動作 |
|---|---|
| タップ | コントロールの表示 / 非表示 |
| ピンチイン / アウト | 拡大・縮小 |
| ダブルタップ | 等倍 ↔ フィット表示のトグル |
| スワイプ左右 | 同フォルダ内の前後ファイルに切り替え |
| 左矢印 / 右矢印ボタン | 同フォルダ内の前後ファイルに切り替え |

スワイプによるページ切り替えは動画ファイルをスキップして画像ファイルのみを対象とする。
漫画ビューア前提の画像フォルダでは、次ページは現在ページの左側に存在するものとして扱う。

---

## トップバー

| 要素 | 位置 | 説明 |
|---|---|---|
| 戻るボタン（←） | 左端 | ファイルブラウザへ戻る |
| ファイル名 | 中央 | 現在表示中のファイル名 |
| 共有ボタン | 右端 | iOS標準のシェアシートを表示 |

## 画像の読み込み

ネットワーク経由のため、段階的に表示する。

```
1. サムネイルキャッシュがあれば先にぼかして表示（プレースホルダー代わり）
2. バックグラウンドで元画像を非同期ダウンロード
3. ダウンロード完了後にフェードインで切り替え
```

大きなサイズの画像はダウンロードに時間がかかる場合があるため、
読み込み中はプログレスインジケーターを表示する。

---

## サムネイル生成（画像の場合）

動画と異なり、画像のサムネイル生成は画像自体をリサイズするだけでよい。

```
1. ネットワークサービスから画像の先頭バイト（最大500KB）を取得
2. UIImage / CGImageSourceCreateWithData でデコード
3. 320×320pt にリサイズしてサムネイルとして保存
```

動画よりも高速に生成できるため、ディレクトリ表示時にほぼ即座にサムネイルを表示できる。

---

## 視聴履歴との連携

画像は再生位置の概念がないため、WatchHistory の `lastPositionSeconds` は `0.0` 固定で記録する。
ただし「最近見た画像」として履歴一覧に表示する。

---

## メディアリストとの連携

画像ファイルはメディアリストに登録できる。
リスト内の画像をタップすると画像ビューアーで開く。

---

## 画面遷移への追加

```
ディレクトリブラウザ
  ├── [動画選択] → 動画プレイヤー（fullScreenCover）
  └── [画像選択] → 画像ビューアー（fullScreenCover）
```

画像ビューアー内でスワイプすると同フォルダ内の画像を順に閲覧できる。
動画は画像ビューアーのスワイプ対象から除外する（動画はタップで個別に開く）。
