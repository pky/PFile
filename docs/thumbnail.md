# サムネイル生成・キャッシュ設計

---

## 生成方針

SMB経由のファイルはURLスキームで直接AVAssetImageGeneratorに渡せないため、
ファイルの先頭部分を一時ファイルに書き出してからサムネイルを生成する。

---

## 生成フロー

```
1. AMSMb2でファイルの先頭2MBを読み込む
2. NSTemporaryDirectory() に一時ファイルとして書き出す
3. AVURLAsset で一時ファイルを開く
4. AVAssetImageGenerator でサムネイルを生成
   - 生成位置: 再生時間の10%地点（取得失敗時は5秒地点にフォールバック）
   - サイズ: 320×180pt（@2x / @3xはOSが自動対応）
5. 一時ファイルを削除
6. 生成した画像をキャッシュに保存して返却
```

---

## 2段階キャッシュ

```
リクエスト
  │
  ▼
[メモリキャッシュ（NSCache）]
  ヒット → 即返却
  ミス ↓
[ディスクキャッシュ]
  ヒット → メモリにも載せて返却
  ミス ↓
[サムネイル生成]
  生成後 → メモリ・ディスク両方に保存して返却
```

### メモリキャッシュ

| 項目 | 値 |
|---|---|
| 実装 | NSCache |
| キー | SHA256(nasConfigId + filePath) |
| 最大エントリ数 | 100件 |
| 最大メモリ使用量 | 50MB |

メモリ警告（`UIApplication.didReceiveMemoryWarningNotification`）を受信したら全クリア。

### ディスクキャッシュ

| 項目 | 値 |
|---|---|
| 保存場所 | `Library/Caches/thumbnails/` |
| ファイル名 | SHA256(nasConfigId + filePath).jpg |
| 有効期限 | 7日間（最終アクセス日時で管理） |
| 最大容量 | 500MB |
| 削除方式 | 容量超過時にLRU（最も古くアクセスされたものから削除） |

`Library/Caches/` はOSによる自動削除対象のためバックアップ不要。

---

## 生成の優先制御

### スクロール中の制御

スクロール中はサムネイル生成を一時停止し、停止後に再開する。
`ScrollView` のスクロール状態を検知して `ThumbnailService` に通知。

### 重複リクエストの排除

同一ファイルへの重複生成リクエストはdeduplication（進行中のTaskをキーで管理）。
2つ目以降のリクエストは最初のTaskの完了を待って同じ結果を返す。

### 生成優先度

- 画面に表示されているセル: 高優先度（`.userInitiated`）
- 画面外のプリフェッチ: 低優先度（`.background`）

---

## プレースホルダー

サムネイル未生成・生成失敗時は動画アイコン（SF Symbols: `film`）を表示。
生成完了後はフェードイン（duration: 0.2秒）でサムネイルに切り替える。

---

## サムネイル更新

一度キャッシュされたサムネイルは7日間有効。
ファイルの更新日時（modifiedAt）がキャッシュ保存日時より新しい場合は再生成する。
