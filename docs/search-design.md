# 検索機能設計

## 目的

- 現在表示中の一覧から目的の項目を素早く見つけられるようにする
- 既存の並び替え、表示形式、複数選択、再生導線を崩さずに追加する
- 初期版は安全性を優先し、ブラウズ画面専用の検索シートに限定する

## 対象範囲

### 初期実装に含める

- ブラウズ画面の現在フォルダ内検索
- 検索ボタンから開く専用検索シート
- ファイル名の部分一致検索
- 大文字小文字を区別しない絞り込み
- 入力中のリアルタイム反映
- 検索結果0件時の専用 empty state
- 検索結果からのリスト追加

### 初期実装に含めない

- 接続先を横断したグローバル検索
- サブフォルダを含む再帰検索
- SMB サーバーへの検索クエリ送信
- タグ検索、日付検索、サイズ検索
- 検索条件の永続化
- リストタブ内の検索

## 要件定義

### 1. ブラウズ画面

- 対象画面: ディレクトリブラウザ
- 対象範囲: 現在表示中の `items`
- 検索対象項目: `DirectoryItem.name`
- 検索中は `リスト未登録のみ` を無視すること
- 検索によってディレクトリ再取得は発生させないこと

### 2. 共通要件

- 検索欄は検索シート内の `.searchable` を使う
- 空文字のときは現在フォルダの全件表示に戻る
- 一致0件のときは「データなし」とは区別した文言を出す
- 検索状態はシート内状態として持ち、閉じるとリセットされる

## UI仕様

### ブラウズ画面

- 右上に検索ボタンを置く
- ボタン押下で検索シートを開く
- シート内に `.searchable` を置く
- プレースホルダ: `このフォルダ内を検索`
- 結果はリスト表示で出す
- 検索シート内で複数選択し、`リストに追加` を実行できる

### 0件時表示

- タイトル: `一致する項目がありません`
- 補足: `検索条件を変更してください`

## 実装方針

### 状態管理

- 検索文字列は検索シートの `@State` で持つ
- 検索結果は `FileBrowserViewModel.items` からその場で絞り込む
- 通常一覧の未登録フィルタ状態とは切り離す

### 検索アルゴリズム

- `localizedCaseInsensitiveContains` を使用する
- 前後空白は `trimmingCharacters(in: .whitespacesAndNewlines)` で除去する
- 空文字は現在フォルダの全件表示として扱う

## テスト方針

- 空文字で全件表示になること
- 部分一致で絞り込めること
- 大文字小文字を区別しないこと
- `リスト未登録のみ` の ON/OFF に関係なく検索結果が変わらないこと
- 検索結果からリスト追加できること

## 将来拡張

- 再帰検索は別機能として設計を分ける
- 実装する場合は、画面内検索とは別に専用 ViewModel とキャンセル可能な `Task` を持たせる
- グローバル検索を入れる場合は、現在の一覧検索と混ぜずに独立した検索画面にする
