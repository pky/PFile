# ホーム画面 / サイドバー 設計

縦持ち（メイン）ではホーム画面、横持ちではサイドバーとして同じ構成を表示する。

---

## レイアウト構成

```
┌─────────────────────────────┐
│  ネットワーク           [+] │  ← セクションヘッダー + 新規接続ボタン
│    📁 サンプルNAS               │
│    📁 会社NAS               │
│                             │
│  ローカル               [+] │
│    📁 Downloads             │
│    📁 Anime Cache           │
│    📷 フォトライブラリ       │
│                             │
└─────────────────────────────┘
```

---

## セクション詳細

### ネットワーク

- 保存済みのNAS接続先を一覧表示
- セクションヘッダー右端に [+] ボタン → NAS新規接続画面をシートで表示
- 各NASをタップ → 接続してディレクトリブラウザへ遷移
- 各NASを長押し or スワイプ → 編集・削除メニュー
- 接続中のNASにはインジケーター（接続中スピナーまたは接続済みバッジ）を表示

### ローカル

- PFile が端末全体のローカルストレージを自由に走査するのではなく、ユーザーが `Files` から明示的に選んだフォルダだけを表示する
- セクションヘッダー右端に [+] ボタン → フォルダ選択を開き、追加済みローカルフォルダとして保存する
- 追加済みフォルダをタップ → そのフォルダを起点にローカルブラウザへ遷移
- 追加済みフォルダは長押し or スワイプで削除可能
- フォトライブラリは固定項目として表示する
- `私のiPadで` のような疑似ルートは作らない

### ローカルをこの形にする理由

- 現時点の PFile にはファイル移動・コピー・インポート機能がなく、「PFile 内に動画を集める場所」として見せるのは実態とズレる
- そのため、ローカル機能は「既に Files にあるフォルダを参照する補助機能」として扱う
- PFile の主軸は NAS 接続であり、ホーム画面でもネットワークを主導線に置く

## 縦持ち（ホーム画面）と横持ち（サイドバー）の出し分け

| 向き | 表示形式 |
|---|---|
| 縦持ち | ルート画面としてフル表示。ネットワーク接続または追加済みローカルフォルダを選ぶとディレクトリブラウザへ push 遷移 |
| 横持ち | `NavigationSplitView` の左カラム（サイドバー）として表示。選択状態を維持したまま右カラムにディレクトリが表示される |

実装上は同一の View を共用し、`NavigationSplitView` と `NavigationStack` で包む部分だけ切り替える。

## NAS新規接続画面（シート）

2段階の入力フローで構成する。

### ステップ1: 接続情報の入力

| 項目 | 例 | 備考 |
|---|---|---|
| 表示名 | サンプルNAS | 任意の名前 |
| ホスト / IPアドレス | nas.example.local | 必須 |
| ポート | 445 | デフォルト値を自動入力 |
| 共有フォルダ名 | videos | 必須 |
| ユーザー名 | admin | ゲスト接続の場合は空欄でも可 |
| パスワード | ••••• | Keychainに保存 |

入力後「接続」ボタンをタップ → NASへ接続を試みる。
- 接続失敗 → エラー内容をシート内に表示（タイムアウト / 認証失敗 / ホスト不明）

### ステップ2: トップフォルダの選択

接続成功後、同シート内でフォルダ選択画面に切り替わる。

```
┌─────────────────────────────┐
│  トップフォルダを選択        │
│  （スキップで共有ルートを使用）│
├─────────────────────────────┤
│  📁 /                  ← ルート（現在地）
│  📁 Movies                  │
│  📁 TV                      │
│  📁 Anime                   │
│  📁 Downloads               │
└─────────────────────────────┘
         [ここを使う]
```

- フォルダを選んで中に入ることができる（パンくず表示あり）
- 「ここを使う」ボタンで現在表示しているフォルダをトップに設定
- 「スキップ」でルート（`/`）をトップに設定
- 設定後、シートを閉じてNAS一覧に追加

### 接続済みNASの編集

NASを長押し → 「編集」でステップ1の内容を編集できる。
トップフォルダも編集画面から変更可能（再接続してフォルダ選択フローに戻る）。

---

## ローカルフォルダ追加

- 追加ボタン文言は `ローカルフォルダを追加` とする
- 空状態では `ローカルフォルダはまだ追加されていません` と表示する
- 補助文言は `「ローカルフォルダを追加」から選んだフォルダが、ここに表示されます。` とする
- ローカル詳細画面のタイトルは疑似ルート名ではなく、選択したフォルダ名そのものを表示する
