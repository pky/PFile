# ファイル操作設計

---

## 対応操作一覧

| 操作 | 単体 | 複数選択 |
|---|---|---|
| アップロード | ✓ | ✓（複数ファイル同時） |
| ダウンロード | ✓ | ✓ |
| 移動 | ✓ | ✓ |
| コピー | ✓ | ✓ |
| 削除 | ✓ | ✓ |
| リネーム | ✓ | - |
| フォルダ作成 | ✓ | - |

---

## UI操作のトリガー

### 単体操作：長押しコンテキストメニュー

ファイル・フォルダを長押しするとコンテキストメニューを表示する。

```
📁 フォルダ名 または 🎬 ファイル名
─────────────────────
  ダウンロード
  移動
  コピー
  リネーム
  ─────────────────
  削除          ← 赤色・破壊的操作
```

フォルダの場合は「ダウンロード」を「フォルダをまとめてダウンロード」に変更。

### 複数選択：選択モード

ナビゲーションバーに「選択」ボタンを追加。タップで選択モードに入る。

```
┌─────────────────────────────────────┐
│  ← サンプルNAS    [3件選択中]    完了   │  ← 選択モード時のナビゲーションバー
├─────────────────────────────────────┤
│  ☑ 📁 アクション                    │
│  ☐ 📁 SF                           │
│  ☑ 🎬 サンプル動画A.mp4               │
│  ☑ 🎬 サンプル動画B.mkv               │
├─────────────────────────────────────┤
│ [ダウンロード] [移動] [コピー] [削除]│  ← ツールバー（選択数が0の場合はグレーアウト）
└─────────────────────────────────────┘
```

- 「完了」で選択モードを解除
- 「すべて選択」はナビゲーションバーの「選択中」ラベルをタップで切り替え

## 各操作の詳細仕様

### アップロード

トリガー: ナビゲーションバーの [+] メニュー → 「ファイルをアップロード」または「フォルダを作成」

```
[+] メニュー
  ├── 写真・動画を選択（フォトライブラリ）
  ├── ファイルを選択（iOS Files アプリ）
  └── フォルダを作成
```

- `PHPickerViewController` でフォトライブラリから選択
- `UIDocumentPickerViewController` でFilesアプリから選択
- 複数ファイル同時選択・アップロード対応
- アップロード中はプログレスシートを表示（後述）

### ダウンロード

- 動画・画像: フォトライブラリに保存（`PHPhotoLibrary`）
- その他ファイル: Files アプリの所定フォルダに保存（`UIDocumentPickerViewController`）
- ダウンロード中はプログレスシートを表示

### 移動・コピー

フォルダ選択ピッカーを表示して移動・コピー先を選ぶ。

```
┌─────────────────────────────┐
│  移動先を選択               │
│  ← サンプルNAS / Videos        │
├─────────────────────────────┤
│  📁 アクション              │
│  📁 SF                      │
│  📁 ドラマ                  │
├─────────────────────────────┤
│  [ここへ移動]               │
└─────────────────────────────┘
```

- 同一サービス内の移動・コピーを扱う
- 異なるサービス間のコピーは扱わない
- 移動先に同名ファイルが存在する場合は競合ダイアログを表示（後述）

### 削除

確認アラートを必ず表示する。

```
「サンプル動画A.mp4」を削除しますか？
この操作は取り消せません。

  [キャンセル]  [削除]  ← 削除は赤色
```

複数選択時: 「3件のファイルを削除しますか？」

削除成功後:
- ファイルブラウザの一覧から即時除去
- 対応するメディアリスト・視聴履歴は残す（接続先が見つからない旨を表示）

### リネーム

インラインテキストフィールドまたはアラートで入力。

```
名前を変更
──────────────────
  [サンプル動画A        ]  ← 拡張子は除いた状態で表示・編集
  　　　　　　　　　　   保存時に拡張子を自動で付け直す
  [キャンセル]  [保存]
```

- 拡張子の変更は不可（UI上で拡張子を非表示にして保護）

### フォルダ作成

```
フォルダ名を入力
──────────────────
  [新しいフォルダ   ]
  [キャンセル]  [作成]
```

## プログレス表示

アップロード・ダウンロードなど時間のかかる操作はプログレスシートを表示する。

```
┌─────────────────────────────┐
│  アップロード中             │
│                             │
│  サンプル動画A.mp4              │
│  ████████░░░░░░  60%       │
│  残り約 30秒                │
│                             │
│  2 / 3 ファイル             │
│                             │
│           [キャンセル]      │
└─────────────────────────────┘
```

- 複数ファイルの場合は全体の進捗と現在のファイル名を表示
- バックグラウンド移行時も転送を継続（URLSessionのバックグラウンド転送を使用）
- キャンセル時は転送中のファイルは破棄し、完了済みのファイルはそのまま保持

---

## 競合ダイアログ（同名ファイルが存在する場合）

```
「サンプル動画A.mp4」はすでに存在します

  [置き換える]
  [両方残す]      ← コピー先に「サンプル動画A (2).mp4」として保存
  [スキップ]      ← 複数ファイル操作時のみ表示
  [キャンセル]
```

---

## エラーハンドリング

| エラー | 表示 |
|---|---|
| ネットワーク切断 | 操作を中断してアラート表示。完了済み分は保持 |
| 容量不足（デバイス） | 「デバイスの空き容量が不足しています」 |
| 容量不足（NAS側） | 「接続先の空き容量が不足しています」 |
| 権限なし | 「このフォルダへの書き込み権限がありません」 |
| ファイル使用中 | 「ファイルが使用中のため操作できません」 |

## FileRepository Protocol への追加メソッド

```
protocol FileRepository {
  // 既存（読み取り）
  func listDirectory(path: String) async throws -> [DirectoryItem]
  func readFile(path: String, range: Range<Int64>?) async throws -> Data

  // 追加（書き込み）
  func uploadFile(localURL: URL, remotePath: String, progress: ((Double) -> Void)?) async throws
  func downloadFile(remotePath: String, localURL: URL, progress: ((Double) -> Void)?) async throws
  func moveItem(from: String, to: String) async throws
  func copyItem(from: String, to: String) async throws
  func deleteItem(path: String) async throws
  func renameItem(path: String, newName: String) async throws
  func createDirectory(path: String) async throws
}
```
