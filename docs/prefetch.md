# 動画先読み設計

---

## 目的

SMB経由のネットワーク遅延を吸収し、動画を選択してから再生開始までの待機時間をゼロに近づける。

---

## 先読みの方式

動画ファイル全体のダウンロードは行わない。
`AVURLAsset` の非同期プロパティロード（`.isPlayable` / `.duration`）のみ実行し、
再生準備が整った状態のAssetをキャッシュする。

実際の映像データはストリーミング（Range Request相当）で逐次取得する。

---

## 先読み対象

ディレクトリブラウザで現在表示されているリストのうち、
スクロール方向の先頭から3件の動画ファイルを対象とする。

- スクロール停止後 0.5秒後に先読みを開始（スクロール中は実行しない）
- リストが更新（ディレクトリ移動・並び替え）されたら先読みキャッシュをリセット

---

## PrefetchManager

### AVAssetキャッシュ

| 項目 | 値 |
|---|---|
| キャッシュ最大件数 | 5件 |
| キー | filePath |
| 保持期間 | ディレクトリを離れるまで |

5件を超えた場合は最も古いエントリを破棄する。

### 動画選択時の動作

```
[ユーザーが動画を選択]
  │
  ├─ キャッシュあり → キャッシュ済みAVAssetをAVPlayerに渡して即再生
  │
  └─ キャッシュなし → AVAssetを新規生成してAVPlayerに渡す（通常フロー）
```

---

## SMBストリーミング

AMSMB2の `SMB2FileHandle` を使い、AVAssetのResourceLoaderに実装する。
再生に必要なバイト範囲のみをSMBから逐次取得することで、全体ダウンロードを回避する。

```
AVPlayer → ResourceLoader → SMB2FileHandle → NAS
              （Range Request相当でバイト範囲指定読み込み）
```

---

## 先読みの停止条件

以下の状況では先読みを停止・キャンセルする。

| 状況 | 対応 |
|---|---|
| スクロール中 | 先読み開始を保留（停止後に再開） |
| アプリがバックグラウンドに移行 | 進行中の先読みをキャンセル |
| ネットワーク低速・圏外 | 先読みを停止（NWPathMonitorで検知） |
| ディレクトリ移動・並び替え | キャッシュリセット・先読みやり直し |

### ネットワーク状態の監視

`Network.framework` の `NWPathMonitor` でネットワーク状態を監視。

| 状態 | 先読み |
|---|---|
| Wi-Fi / 有線LAN | 有効 |
| 低速ネットワーク | 停止 |
| 圏外 | 停止 |

---

## 同時実行数

先読みの同時実行数は最大2に制限する。
それ以上は待機キューに積み、順次実行する。
