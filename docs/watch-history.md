# 視聴履歴機能設計

---

## 概要

動画を再生した記録を自動的に保存する。
同一ファイルは1レコードとして管理し、最終視聴位置を更新し続ける。
次回そのファイルを開いたとき、最後に見た位置から再生できる。

---

## 保存タイミング

| タイミング | 処理 |
|---|---|
| 再生開始から10秒後 | 初回レコード保存（誤タップ防止のため10秒の猶予を設ける） |
| 再生中30秒ごと | `lastPositionSeconds` と `watchedAt` を更新 |
| アプリがバックグラウンドに移行 | 即時保存 |
| プレイヤーを閉じる | 即時保存 |

---

## upsertのルール

保存キー: `nasConfigId + filePath` の組み合わせ。

- レコードが存在しない → 新規INSERT
- レコードが存在する → `lastPositionSeconds` と `watchedAt` をUPDATE

同じ動画を再度開いても履歴が重複しない。

---

## 視聴完了の判定

総時間の95%以上まで再生した場合を「視聴完了」とみなす。
視聴完了した動画を次回開く場合は先頭（0秒）から再生する。

---

## 視聴履歴一覧画面

### 表示内容

```
┌───────────────────────────────────┐
│ [サムネイル] タイトル.mp4          │
│             01:23:45 / 02:00:00   │
│             2026/02/20 視聴        │
└───────────────────────────────────┘
```

- サムネイル: ThumbnailServiceのキャッシュを参照
- 進捗バー: `lastPositionSeconds / durationSeconds` の割合をバーで表示
- 視聴日時: `watchedAt` を相対表示（「3時間前」「昨日」など）

### 並び順

`watchedAt` 降順（最近見たものが上）。固定。

### 操作

- タップ: 最終視聴位置から再生
- スワイプ左: 履歴を削除
- 長押し: コンテキストメニュー（削除のみ）

---

## 履歴件数の管理

最大保持件数: 500件。
501件目が追加されたタイミングで最も古い `watchedAt` のレコードを自動削除する。

---

## 再生再開の起動位置の優先順位

同一ファイルをブラウザ・メディアリスト・履歴の複数経路から開く場合がある。
再生開始位置の優先順位は以下の通り。

```
1. 視聴履歴からの起動       → WatchHistory の lastPositionSeconds
2. メディアリストからの起動 → WatchHistory の lastPositionSeconds（履歴があれば）
3. ブラウザからの起動       → WatchHistory の lastPositionSeconds（履歴があれば）
                           履歴がなければ 0秒（先頭）
```

---

## バックグラウンド保存の実装

バックグラウンド移行時（`scenePhase == .background`）に即時保存する。
SwiftDataのバックグラウンドコンテキストを使用してメインスレッドをブロックしない。
